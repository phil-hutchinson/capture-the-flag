# Peer Review — Core game engine and random-play runner

## Summary

This branch implements the full Capture the Flag ruleset as a `game-engine-core`-compatible
engine (board/piece geometry, legal move generation, combat resolution, `apply_ply`
with clocks and the breachability cache, and every ending) plus the random-placement
seam, a `CtfPlayer`/random-player/match wrapper, a game-record writer, and a headless
batch-runner application, closing out all 13 implementation-plan steps. Test coverage
is thorough and closely tracks the rulebook section by section, including the special
combat cases and ending edge cases; a 3000-game acceptance batch ran with no errors.
`pyright` reports 0 errors/warnings and `ruff check .` reports no findings — both clean.

A second review pass augmented this document with findings #3–#7 (re-running `pyright`
and `ruff check .`, both still clean); the original findings #1–#2 and their numbering
are preserved unchanged. The most significant addition is #3, a reproducible
`StopIteration` crash in `apply_ply` on the same Archer-behind-the-Flag geometry that
finding #1 flagged as a rules question.

Finding #8 was added later, during resolution, at the developer's request: the
record file format did not record which ruleset version a game was played under.
